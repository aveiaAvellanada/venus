# SP-4: Cierre de Caja (Modo Manual)
**Execution Plan**

REQUIRED SUB-SKILL: Use `superpowers:subagent-driven-development` to implement this plan task-by-task.

## Contexto
Este plan construye el Módulo de Caja (Apertura y Cierre diario de turnos) exigiendo conteo físico y calculando diferencias. Implementa bloqueos en las ventas si la caja no está abierta.

---

## Task 1: Funciones RPC de Agregación Diaria

Para que ningún dispositivo calcule mal los totales por culpa de cachés o mala conexión, dejaremos que la base de datos sume todo.

**Files:** `supabase/migrations/<ts>_sp4_rpc_caja.sql`

- [x] **Step 1:** Crear la migración SQL con una función `public.obtener_resumen_dia(p_fecha date)` que sume todas las ventas y sus métodos de pago de esa fecha y devuelva un JSON con: `total_ventas` (cantidad), `total_general`, `total_efectivo`, `total_nequi`, `total_daviplata`.
- [x] **Step 2:** Aplicar migración remotamente (vía `npx supabase db push` o CLI vinculada).
- [x] **Step 3:** Regenerar tipos de TypeScript (`npx supabase gen types typescript`).
- [x] **Step 4:** Commit.

---

## Task 2: Capa de Servicios `lib/caja.ts`

**Files:** `lib/caja.ts`

- [ ] **Step 1:** Crear `lib/caja.ts`.
- [ ] **Step 2:** Función `obtenerCajaHoy()`: Busca si hay un registro en `cierres_caja` con la fecha actual (formato 'YYYY-MM-DD'). Retorna el estado ('abierto', 'cerrado', o null).
- [ ] **Step 3:** Función `abrirCaja()`: Crea el registro del día con `estado='abierto'`, `modo='manual'`, `fecha=hoy()`.
- [ ] **Step 4:** Función `obtenerResumenEnVivo()`: Llama a la RPC creada en Task 1.
- [ ] **Step 5:** Función `cerrarCaja(efectivo_contado, diferencia, nota)`: Actualiza la fila de hoy seteando todos los totales (llamando al resumen de forma segura) y cambiando el `estado` a `'cerrado'`.
- [ ] **Step 6:** Typecheck y Commit.
